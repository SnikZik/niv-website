# 05 — צ'קליסט נגישות (תקן ישראלי 5568 / WCAG 2.0 AA)

חובה חוקית באתר ישראלי מסחרי. הנגישות נבנית מההתחלה, לא מולבשת בסוף.

## מבנה סמנטי
- [ ] HTML סמנטי: `<header> <nav> <main> <article> <section> <footer>`.
- [ ] היררכיית כותרות תקינה — H1 יחיד, בלי דילוגי רמות.
- [ ] `lang="he"` + `dir="rtl"` על `<html>`.
- [ ] קישור "דלג לתוכן" (skip-to-content) ראשון ב־DOM, נראה ב־focus.

## אינטראקציה ומקלדת
- [ ] כל פעולה נגישה במקלדת (Tab/Enter/Esc).
- [ ] מצב focus **נראה** (outline ברור, לא `outline:none`).
- [ ] כפתורים = `<button>` אמיתי; קישורים = `<a>` עם href אמיתי.
- [ ] אקורדיון FAQ: `aria-expanded`, `aria-controls`, ניתן להפעלה במקלדת.
- [ ] CTA דביק לא חוסם תוכן ולא לוכד focus.

## טפסים
- [ ] `<label>` מקושר לכל שדה (`for`/`id`).
- [ ] שגיאות ולידציה ברורות, מקושרות (`aria-describedby`), לא רק צבע.
- [ ] `autocomplete` מתאים (tel, name).

## מדיה וצבע
- [ ] alt משמעותי לתמונות תוכן; `alt=""` לדקורטיביות.
- [ ] ניגודיות: טקסט על רקע ≥ 4.5:1. **בדיקה קריטית:** אדום `#C0392B` על לבן = ~4.9:1 ✓ ; פחם `#24211F` על חול = מצוין ✓ ; להימנע מ־אבן `#C6A16B` כטקסט על לבן (ניגודיות נמוכה — לאקסנט בלבד).
- [ ] אין הסתמכות על צבע בלבד להעברת מידע.
- [ ] אין אודיו/וידאו בהפעלה אוטומטית.
- [ ] כיבוד `prefers-reduced-motion` — ביטול אנימציות.

## נייד ו־RTL
- [ ] RTL תקין בכל רכיב (padding/margin לוגיים: `margin-inline`, `padding-inline`).
- [ ] גודל פונט קריא בנייד (בסיס ≥ 16px).
- [ ] יעדי מגע ≥ 44×44px (כפתורי חיוג).

## תשתית
- [ ] עמוד **הצהרת נגישות** (`/הצהרת-נגישות/`) — כולל פרטי רכז נגישות `NEEDS_CLIENT_INPUT`.
- [ ] רכיב נגישות (תוסף כמו One / Nagich) — אופציונלי, לא תחליף לנגישות אמיתית בקוד.
- [ ] בדיקה: Lighthouse a11y ≥ 95, ניווט מקלדת מלא, קורא מסך (VoiceOver) על הבית + עמוד שירות.
